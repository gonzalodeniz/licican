# Roadmap de PodencoTI

## Criterios de planificacion
- Se prioriza validar primero la propuesta de valor central: centralizacion y filtrado de oportunidades TI.
- Se evita comprometer cobertura total de todas las administraciones en la primera entrega.
- Las fases representan releases funcionales, no decisiones tecnicas de implementacion.

## Release 0: Definicion funcional del MVP
- Objetivo: Cerrar ambiguedades criticas antes de la implementacion.
- Alcance:
  - PB-007 Cobertura inicial de fuentes prioritarias.
  - PB-006 Reglas funcionales de clasificacion TI.
  - Definicion de criterios de aceptacion base para catalogo, detalle y filtros.
- Resultado esperado:
  - `developer-teams` recibe alcance funcional acotado y validable.

## Release 1: MVP de descubrimiento
- Objetivo: Permitir descubrir y evaluar oportunidades TI en un unico lugar.
- Alcance:
  - PB-001 Catalogo inicial de oportunidades TI de Canarias.
  - PB-002 Ficha de detalle de licitacion.
  - PB-003 Filtros funcionales de busqueda.
- Resultado esperado:
  - El usuario puede consultar, revisar y filtrar oportunidades sin recorrer manualmente varios portales.

## Release 2: MVP de seguimiento y alertas
- Objetivo: Convertir el descubrimiento en uso recurrente y gestion operativa.
- Alcance:
  - PB-004 Configuracion de alertas tempranas.
  - PB-005 Pipeline de seguimiento de oportunidades.
- Resultado esperado:
  - El usuario recibe valor recurrente y puede gestionar su trabajo sobre licitaciones detectadas.

## Release 3: Medicion y optimizacion
- Objetivo: Mejorar precision, cobertura y aprendizaje de producto.
- Alcance:
  - PB-008 Medicion basica de valor del producto.
  - Ajustes de priorizacion segun feedback y resultados de QA.
- Resultado esperado:
  - Existe una base de decision para ampliar cobertura y evolucion funcional.

## Dependencias abiertas de roadmap
- Confirmar el subconjunto exacto de fuentes oficiales que entra en MVP.
- Confirmar el criterio minimo para considerar una licitacion como TI relevante.
- Alinear con `qa-teams` el enfoque de validacion funcional para Release 1 y Release 2.
